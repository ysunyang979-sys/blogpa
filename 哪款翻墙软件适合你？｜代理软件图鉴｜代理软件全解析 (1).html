一. 全民级主力（目前用户最多）


clash veger Rev  (Win/Mac/Linux)    点击下载

  • 全名： Clash Verge Rev (Revolution)。
  • 血统： 它是原版 Clash Verge 的续作。
  • 核心亮点： 它是基于 Rust 语言和 Tauri 框架开发的。

    它比老掉牙的 CFW 更轻量、占用内存更低、打开速度更快，而且界面非常现代，符合现在的审美。

内置 Mihomo 内核

  这是 Verge 强过其他软件的核心原因。它内置了 Mihomo 内核（也叫 Clash Meta）。

  • 协议支持： 2026 年最火的 Reality、Hysteria2、TUIC v5，它全都能跑。
  • 分流逻辑： 支持更复杂的规则判断，比如根据 App 进程分流，或者根据网站的证书信息分流。

核心功能

① 订阅管理（Profiles）

一键导入： 直接粘贴订阅链接。

增强脚本（Merge / Script）： 这是高阶功能。可以一键修改订阅里的节点名字、自动添加广告过滤规则等。

② TUN 模式（虚拟网卡）

开启后，全电脑的流量（包括游戏、终端、微软商店）都会走代理。不再需要手动在每个软件里设置代理了。

③ 策略组可视化

它的界面非常直观，你可以点击不同的“策略组”（比如“自动选择”、“香港节点”、“美国节点”），让不同的网站走不同的出口。

④ CSS 注入（颜值控最爱）

允许用户自己写点代码改界面颜色、背景。

优缺点对比

优点缺点
界面美： 天花板级别的 UI。有学习成本： 规则和脚本功能需要花点时间研究。
内核强： Mihomo 内核更新极快，支持新技术。兼容性： 极个别老旧系统（如 Win 7）运行可能不稳。
全平台统一： Win、Mac、Linux 操作逻辑几乎一样。

V2rayN （Win/Mac/Linux）    点击下载
  • 全名： v2rayN
  • 地位： Windows 平台上历史最悠久、更新最勤快、用户基数最大的 GUI 客户端。
  • 核心逻辑： 它是典型的**“外壳软件”**。它本身不处理流量，而是通过调用底层的“内核”（如 Xray、Sing-box）来工作。

如果把代理协议比作引擎，v2rayN 就是那个能装下所有引擎的超级底盘。

多内核驱动（Multi-Core）

这是 v2rayN 最大的卖点。它不绑定某一个内核，而是允许用户自由切换：

  • Xray-core： 玩 VLESS + Reality 的首选，最正宗、最稳定。

  • Sing-box： 性能王者，跑 Hysteria2 和 TUIC v5 速度极快。

  • V2Fly / Clash： 兼容老旧协议和特殊的规则格式。

核心功能

① 协议“全家桶”

这是 v2rayN 傲视群雄的地方。不管是十年前的 Shadowsocks，还是去年刚火的 Reality，或者是今年最新的协议，v2rayN 总是第一批支持的。

② 强大的“右键菜单”

演示时可以重点展示任务栏图标的右键功能：

  • 自动测速： 一键测试所有节点的延迟和速度。

  • 订阅更新： 逻辑非常简单，直接把订阅链接一扔即可。

③ TUN 模式（全系统接管）

虽然早期的 v2rayN 在全系统代理上不如 Clash，但现在的版本已经非常完善。

  • 演示重点： 开启 TUN 模式后，即便是没有代理设置的命令行工具、UWP 应用或者是轻量级网游，都能轻松“出海”。

④ 节点分享与导入

它的二维码生成和剪贴板导入极其方便，非常适合互助分享节点。

优缺点对比



Karing （Win / Mac / Android / iOS / TV）点击下载

  • 定位： 基于 Flutter 开发的新一代全平台、规则驱动代理工具。

  • 核心逻辑： 它的目标是让用户在 Windows、安卓和 iPhone 上看到完全一样的界面，用完全相同的操作逻辑。

  • 血统： 它是一个“缝合怪”（褒义词），它把 Clash 的规则逻辑、Sing-box 的极致性能和苹果系的审美完美融合在了一起。

核心功能与亮点

① 恐怖的“订阅兼容性”

这是小白最爱的功能。不管你手里的订阅链接是 Clash 格式、V2ray 格式、Stash 格式、还是 Sing-box 格式，甚至是普通的 SS 订阅，直接往 Karing 里扔，它都能自动识别并转换。

② “多端同步”黑科技

如果你在 Windows 上配置好了分流规则，可以通过 iCloud（苹果系）、WebDAV 或者是 局域网 直接同步到你的手机和电视上。

  •  配置一次，全家受用

③ 极致的 UI 与“新手模式”

  • 它的界面非常“果粉”，甚至比 Stash 还要简洁

  • 它内置了“新手模式”，隐藏掉那些复杂的协议参数，只给用户看“开关”和“节点选择”，极大降低了入坑门槛

④ 底层实力：内置 Sing-box 与 Mihomo

虽然外表斯文，但内心极其暴力。它支持几乎所有尖端协议：

  • Reality / Hysteria2 / TUIC v5 / SSH / WireGuard 统统不在话下

优缺点对比

优点缺点
全平台体验一致： 换设备不需要重新学习。资源占用： 由于是 Flutter 开发，相比原生 Sing-box 内存占用稍高。
内置规则集： 自动更新 GeoIP 和路由规则，非常省心。过于简洁： 某些极端极客的功能（如复杂的脚本重写）可能不如 Surge。
Apple TV 适配： 可能是目前 Apple TV 上最好用的代理软件之一。新秀阶段： 虽然 2026 年很稳，但比起老牌软件，小 bug 修复频率极高。

二. 现代派新势力（颜值高、高性能）


Hiddify （Win/Mac/Android/iOS/Linux/TV）点击下载


  • 定位： 基于 Sing-box 内核的次世代全平台通用代理客户端。
  • 核心哲学： “让代理消失”。它的目标是让用户感知不到协议、内核这些复杂的词汇，只需要点一下“连接”。
  • 设计语言： 采用 Flutter 开发，UI 极度现代化，支持丰富的动画效果和响应式布局。


核心功能与亮点

① 智能自动化（核心卖点）

  • 自动测速与选择： 它不仅仅是显示延迟，还会根据你的网络环境自动切换到延迟最低、带宽最稳的节点。

  • 全格式兼容： 无论粉丝手里的是 Clash 链接、V2ray 链接还是 Sing-box 链接，甚至是直接一个节点的 URL，Hiddify 统统“通吃”。

② 地区预设规则（新手救星）

  • 它内置了针对不同地区（如 China, Iran, Russia）的智能路由预设。

  • 演示重点： 你只需要在设置里选“China”，它就会自动帮你把抖音、微信等国内流量切回直连，不需要你去手动导入复杂的路由规则。

③ 多种模式一键切换

  • VPN 模式 (TUN): 接管系统全流量，适合游戏和全局办公。

  • 系统代理模式: 仅接管浏览器流量。

  • 仅代理模式: 给极客玩家手动配置使用。

④ 可视化数据统计

  • 主页会非常直观地显示你的剩余流量、订阅到期时间以及实时流量波形图，颜值极高。

优缺点对比

Mihomo (Clash Meta) （Win/Mac/Linux）    点击下载


  • 前世今生： 它起初叫 Clash Meta，是原版 Clash 停止更新后，社区为了加入更多新功能（如 Reality、Hy2）而开发的加强版。2024 年后，为了规避商标风险，正式更名为 Mihomo。
  • 核心逻辑： 它本质上是一个 “内核（Kernel）”。你现在看到的 90% 的新一代 Clash 系软件（如 Verge Rev, Stash, Mihomo Party）其实用的都是 Mihomo 的“心”。
  • 总结： 它不是一个简单的软件，它是所有 Clash 系玩家的救命稻草。

核心功能

① 协议“大圆满”

这是它相比原版 Clash 最强的地方。它原生支持最顶级的协议：

  • VLESS + Reality: 完美的网页伪装。

  • Hysteria2 / TUIC v5: 极致的 UDP 加速，网络再破也能跑满带宽。

  • WireGuard: 直接支持主流的加密隧道协议。

② 恐怖的“规则引擎”

  • 逻辑规则： 支持 AND、OR、NOT 逻辑。比如：你可以设置“当流量来自 Telegram 且 目的地是美国 且 当前是晚上”时，才走某个特定节点。

  • 进程分流： 演示如何让“网易云音乐”客户端走代理（解锁灰色歌曲），而浏览器走直连。

③ 链式代理（Relay / 节点跳板）

  • 这是你之前做过的内容！Mihomo 支持将节点 A 和节点 B 串联。

  • 演示重点： 国内机 -> 国外机，实现极致的本地化出口（比如让你的 IP 看起来始终在香港，即便是用中转线路）。

④ 内置可视化面板

  • 它支持连接 MetaCubeXD 等精美的网页版仪表盘。在浏览器里就能看到全屋设备的流量曲线、实时延迟和连接地图。

优缺点对比

优点缺点
全能战神： 目前协议支持最快、最全的 Clash 系内核。学习成本： 如果直接用内核或 YAML 配置文件，新手会觉得像看天书。
分流极准： 支持域名指纹检测和精细的 GEO 规则。版本更迭快： 社区更新太勤快，有时候刚调好的脚本新版本就改了。
开源力量： 是社区生命力最强的代理项目之一。

GUI.for.SingBox （Win/Mac/Linux）点击下载

  • 定位： 它是 GUI.for.Cores 系列的核心作品，专门为 Sing-box 内核打造的图形化管理界面。

  • 设计理念： “壳核分离，模块驱动”。它不是一个简单的 App，而是一个让你可以用图形化方式去“拼装” Sing-box 复杂功能的平台。

  • 目标人群： 觉得 v2rayN 界面不够清晰、又觉得 Karing 自由度不够的进阶玩家。

核心功能与亮点

① 强大的“插件/模版”系统（核心竞争力）

  • 这是它的“杀手锏”。你不需要去手写几百行的 Sing-box JSON 代码。

  • 演示重点： 展示如何一键导入“大神的模版”。它将复杂的配置拆解成了“订阅”、“分流”、“DNS”等模块，你可以像插拔 U 盘一样随时更换其中的一部分。

② 极度透明的日志与测速

  • 它的日志面板非常硬核。你可以实时看到每一个连接走了哪个协议、被哪个规则拦截、延迟是多少。

  • 对于排查网络问题（比如为什么某个游戏连不上），它是所有软件里效率最高的。

③ 真正的“内核级别”控制

  • 它允许你直接管理 Sing-box 的内核版本。你可以随时手动升级到测试版内核，第一时间体验最新的协议特性（比如最新的性能优化补丁）。

④ 现代、冷静的 UI 界面

  • 虽然功能硬核，但界面一点都不丑。它采用的是非常现代的侧边栏布局，深色模式下极具质感，在视频里看起来很有“大厂工具”的高级感。

优点缺点

模块化管理： 方便备份和分享部分配置（比如只分享一个 DNS 模版）。配置繁琐： 相比“一键导入”，它更强调“组装”。

GUI.for.Clash （Win/Mac/Linux）   点击下载

  • 定位： 专门为 Mihomo (Clash Meta) 内核打造的、极其灵活的图形化管理界面。

  • 家族背景： 与 GUI.for.SingBox 同出一门，两者界面逻辑近乎一致，只是“心脏”不同。

  • 核心逻辑： 它把配置文件拆解成了订阅 (Subscriptions)、配置 (Profiles) 和 插件 (Plugins)。这种设计让你可以在不改动原订阅的情况下，通过“插件”或“模版”强行给节点加上你想要的规则。

核心功能

① 独树一帜的“插件系统” (Plugin System)

  • 这是 GfC 最强的地方。你可以给它写简单的 JavaScript 插件来处理流量。

  • 演示重点： 展示如何通过插件自动把所有“延迟高”的节点过滤掉，或者自动给所有节点名称加上国旗图标。这种“对配置文件进行动态手术”的能力，是普通 Clash 软件没有的。

② 彻底的“可视化编辑”

  • 虽然它很硬核，但它把 Mihomo 那些天书般的 YAML 配置文件变成了直观的卡片和按钮。

  • 你可以非常方便地通过图形界面调整 DNS 设置、TUN 模式参数或者是并发请求数量。

③ 极致的响应速度

  • 在 2026 年，很多 Electron 开发的软件会显得臃肿，但 GfC 优化得非常好。切换配置、更新订阅几乎是“瞬发”的，不会有明显的卡顿感。

④ 内置丰富的“外部控制器”支持

  • 演示如何通过它连接到各种漂亮的网页仪表盘（Dashboard），让你像看大片一样监控流量地图和连接曲线。

⑤ 模块（Modules）
为了降低普通用户的门槛，Surge 支持“模块”功能。用户只需点击安装一个链接（Module），即可一键实现去广告、解锁流媒体、优化特定 App 体验等功能，无需手动编写复杂的规则。

    • 只要 Mihomo 内核更新了什么新锐协议（比如最新的 Hysteria 2、TUIC v5、VLESS-Reality 甚至一些实验性功能），它绝对是第一个完美支持的，而且各项参数都可以直接在界面上微调。
    ② 极其强大的“覆写”与“脚本”系统
    • 它内置了非常直观的**“配置覆写（Override）”**功能。
    ③ 现代化的 WebUI 与面板
    • 虽然它是桌面软件，但它的操作面板具有极强的扩展性，支持无缝内嵌各种流行的 Dashboard（比如 Yacd、Metacubexd）。
    ④ 内存与架构优化
    • 原生支持 Apple Silicon (M系列芯片)。虽然它是跨平台框架开发的，但经过几代迭代，它在 Mac 上的资源调度已经非常优秀，看 4K/8K 流媒体时的加解密发热量控制得极好。

    路由器

    Openclash                        点击下载
    • 定位： 运行在 OpenWrt 路由器系统上的 Clash 客户端。
    • 地位： 软路由界最强大的插件，没有之一。它不是装在手机或电脑里的，而是装在你的路由器里的。
    • 现状： 随着 OpenWrt 25.12 稳定版的发布，OpenClash 已经进化得极其成熟。它通过集成最新的 Mihomo (Meta) 内核，完美接管了全家所有设备的流量。
      • 设备无关性： 只要连上这个 WiFi，无论是 iPhone、安卓、甚至是不支持装代理软件的 PS5、Switch、智能电视、扫地机器人，全部自动实现科学上网。
      • 全局掌控： 你只需要在一个地方（路由器后台）维护一份订阅，全家人受益，再也不用一个个设备去更新节点。
    核心功能与黑科技
    ① 满血版 Mihomo 内核
    • 演示亮点： 完美支持 VLESS + Reality、Hysteria2 和 TUIC v5。
    • 在软路由强大的 CPU 加持下，它可以轻松跑满千兆带宽，且延迟极低，是 8K 视频和外服游戏的“物理外挂”。
    ② 智能 DNS 过滤（Fake-IP / Redir-Host）
    • 它能从网关层面解决 DNS 污染。
    • 演示建议：展示它如何通过 SmartDNS 联动，实现国内网站秒开、国外网站精准分流。
    ③ 强大的规则集（Rule Providers）
    • 不同于单机版，OpenClash 可以挂载几十个云端规则集。
    • 场景演示： “视频走香港、学术走美国、游戏走专线、国内走直连”，这套逻辑在 OpenClash 里可以跑得极其丝滑。
    ④ 游戏模式（UDP Full Cone）
    • 对于主机玩家，演示它如何开启 UDP 转发。让 Switch 的 NAT 类型直接变成 Type A，这对于联机党来说是绝对的杀手锏。
